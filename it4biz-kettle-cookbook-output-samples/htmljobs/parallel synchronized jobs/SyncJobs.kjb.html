By design Jobs can not be run in parallel. 
If you need this, you might start multiple job processes by the operating system
or the Shell job entry.
To synchronize multiple jobs you can use the File job entries.
This Job will wait for two other Jobs (SyncJobs - Example 1 and 2) to be finished.
If you want to test this with Spoon, you need to run the other Jobs after
this Job. Note: In the log view you see enries of all Jobs.
Do other things here after the two Jobs are finished.
Make sure the timing is enough for this job to start and delete the two files
before the other Jobs will be finished. Otherwise you could run into
race conditions.
Note: In our example Job2 will be faster then Job1.
If the Jobs are started parallel this Job entry has no delay.
