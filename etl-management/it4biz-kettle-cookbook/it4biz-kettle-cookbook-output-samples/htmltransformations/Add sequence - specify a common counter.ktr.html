If you do a preview on the Result step you will note that the generated ID is unique.
If you specify another counter name in either of the sequence generator steps, you will get duplicates.

Step "Generate Rows" does not actually produce output. It creates 1000 empty lines and splits them evenly
before they reach the two "Generate ID" steps (step type "sequence generator"). Both sequence generators
do have the same "Counter name" (double click to edit and see the "Counter name" value). That makes the
generated ids unique, even though they are produced by two different sequence generators.
You basically get the same result if you have 2 row generators (both generating 20 lines, with no
actual output) that are linked to two ID sequence generators. Both ID sequence generators have got the
same "counter name" and thus the finally created IDs are unique. The difference to the example above is
that the generated IDs are in order (enumerated from 1..40).
