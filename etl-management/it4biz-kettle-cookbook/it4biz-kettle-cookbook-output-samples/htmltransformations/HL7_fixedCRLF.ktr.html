This sample demonstrates a fix for a wrong format of the data supplier.

The file has LF instead of CR as segment terminators. The HL7 standard defines the segment terminator as "The segment terminator is the last character of every segment. It is always the ASCII CR character (hex 0D)."
see http://www.hl7.org/documentcenter/public_temp_CEC715A0-1C23-BA17-0CA10E6F260CC70A/calendarofevents/FirstTime/Glossary%20of%20terms.pdf

When the sender can not change the format to CR, this sample replaces all LF by CR. 

Note: You can test the error message when you omit the LF/CR replacement. In this case you would get an error message like: 
           "Can't process message of version '2.3 MFI' - version not recognized"

This sample is only for showing the CRLF replacement.
Further processing of the data and Z-Segments is shown in separate samples.
